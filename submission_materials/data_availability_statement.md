# Data Availability Statement

The release-candidate submission package contains the layout CSV files, weather input, terrain input, deformation-candidate tables, joint layout--aiming screening tables, joint SolarPILOT default-aiming bridge tables, joint system evidence summaries, joint direct-promotion audit tables, same-condition baseline-control tables, direct baseline-control audit tables, leave-one-day-out direct hold-out tables, coordinate-provenance sensitivity tables, aiming-proxy tables, PySAM/SolarPILOT numerical-checking tables, all-phase, high-sample, independent-seed, and V12 240k-ray reduced PySolTrace custom-aimpoint outputs, the V11 convergence audit, the V12 bootstrap confidence audit, journal figures, scripts, and SHA-256 reproducibility manifests used in the manuscript. Review-facing CSV copies are included under `supplementary_data/`, including `supplementary_data/joint_layout_aiming_optimizer/` for the integrated layout--aiming screening run, `supplementary_data/joint_solarpilot_default_tables/` for the SolarPILOT default-aiming bridge of the joint layouts, `supplementary_data/joint_system_summary/` for the merged proxy/SolarPILOT/direct evidence table, `supplementary_data/joint_direct_soltrace_tables/` for the reduced direct promotion audit of the joint representatives, `supplementary_data/strong_baseline_direct_soltrace_tables/` for the completed 27-condition same-run strong-baseline direct gate, `supplementary_data/baseline_strengthening_tables/` for the low-complexity control comparison, `supplementary_data/baseline_direct_soltrace_tables/` for the reduced direct baseline-control and hold-out audits, and `supplementary_data/missing20_geometry_sensitivity/` for the 20-coordinate-gap geometry stress test. The active checksum manifest is included under `reproducibility_manifest/manifest.json` and `reproducibility_manifest/MANIFEST.md`.

The source coordinate pool is derived from the MIT-licensed Zenodo/GitHub archive `10.5281/zenodo.16957381`, after audit and removal of the origin placeholder. That DOI is used only as the coordinate-source citation for the older dataset package. The new layout-algorithm and receiver-flux screening package is publicly available as a versioned GitHub release:

https://github.com/poboll/dunhuang-heliostat-flux-benchmark/releases/tag/v0.1.3

The v0.1.3 release is the manuscript-facing package after the Figure 1 and metadata cleanup. A machine-readable Zenodo metadata file is included as `.zenodo.json`, and `submission_materials/zenodo_doi_runbook_v013.md` lists the required release-to-DOI steps. Until the new DOI is issued, the GitHub release is the public versioned package record and the older Zenodo DOI remains only the source-coordinate citation.

The package supports rerunning and auditing the numerical benchmark. It does not include plant SCADA, survey-grade terrain, plant-grade TMY files, calibrated receiver tube-temperature limits, or commercial retrofit data.
